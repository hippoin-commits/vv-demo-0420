# VV Design System Implementation

## 适用范围（本仓库路径）

- 组件来源目录：`src/components/ui`
- 规范文档：`guidelines/Guidelines.md`
- 变量与主题：`src/styles/globals.css`

## 必须执行流程

1. 先判断需求类型：样式微调 / 组件改造 / 新增组件 / 页面组装。
2. 在 `src/components/ui` 检查是否已有来源组件。
3. 若已有来源组件：优先复用或扩展，不重复造轮子。
4. 若需要改已有组件样式：先告知用户将影响的组件范围，再实施变更。
5. 若没有来源组件：先询问用户“新建组件还是映射到现有组件”，并给出推荐映射。
6. 样式实现时严格遵循变量优先级：语义变量(`@theme`) > 基础变量(`:root`) > 禁止硬编码。
7. 完成后输出预览方式，优先给出可直接执行的本地预览命令与核对清单。

## 样式与组件规则

- 颜色、间距、圆角、字号必须来自 `src/styles/globals.css` 变量。
- 优先使用语义类（如 `bg-primary`、`text-text`、`border-border`）。
- 无语义类可用时，使用变量写法（如 `bg-[var(--color-primary)]`、`text-[length:var(--font-size-md)]`）。
- 禁止 `#hex`、`rgb(...)`、Tailwind 默认色（如 `bg-blue-500`）和任意硬编码像素值。
- 同一类组件（如按钮、卡片、输入框）必须保持同一变量映射策略，避免风格漂移。

## 回答输出格式

每次完成任务后，按以下结构反馈：

1. 组件来源检查结果（找到的组件 / 是否复用）
2. 变量映射（颜色、间距、圆角、字号）
3. 变更影响面（哪些组件或页面会同步变化）
4. 预览方式（命令 + 页面路径 + 验收点）

## 预览方式（必须提供）

优先推荐顺序：

1. `npm run dev`（业务页面联调）
2. `npm run storybook`（组件级对比，若项目有 Storybook）
3. 若用户反馈“看不到预览”，改用“组件最小复现页 + 浏览器硬刷新 + 控制台无报错”三步排查。
